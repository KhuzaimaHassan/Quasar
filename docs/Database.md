# Database

## Overview

Quasar uses a single PostgreSQL database for all relational data. The `pgvector` extension is added for vector similarity search (RAG). Redis is used alongside PostgreSQL from M4 onward for the short-term conversation memory buffer.

ORM: **Prisma** (type-safe, migration-based, integrates cleanly with Next.js).

---

## Setup

```bash
# Install Prisma
npm install prisma @prisma/client
npx prisma init

# Enable pgvector (run once on your PostgreSQL instance)
CREATE EXTENSION IF NOT EXISTS vector;

# Run migrations
npx prisma migrate dev --name init
```

### `chunks`

Document fragments with their vector embeddings.

| Column | Type | Notes |
|--------|------|-------|
| `id` | `uuid` PK | |
| `document_id` | `uuid` FK → documents | |
| `chunk_index` | `int` | Ordering within document |
| `content` | `text` | Raw chunk text |
| `embedding` | `vector(1536)` | pgvector column — 1536 for OpenAI, 768 for Cohere |

## Relationships

```
users
  ├── workspaces (1:many)
  │     ├── conversations (1:many)
  │     │     ├── messages (1:many)
  │     │     │     └── message_sources (1:many)
  │     │     └── agent_runs (1:many)
  │     └── documents (1:many)
  │           └── chunks (1:many)
  │                 └── message_sources (referenced by)
  └── memories (1:many)
```

## Indexes

```sql
-- Fast conversation lookup
CREATE INDEX idx_conversations_user ON conversations(user_id);
CREATE INDEX idx_messages_conversation ON messages(conversation_id);

-- pgvector similarity search (IVFFlat — good for < 1M rows)
CREATE INDEX idx_chunks_embedding ON chunks
  USING ivfflat (embedding vector_cosine_ops)
  WITH (lists = 100);

-- Memory lookups
CREATE INDEX idx_memories_user_scope ON memories(user_id, scope);

-- Document status polling
CREATE INDEX idx_documents_status ON documents(status);
```

> Switch to HNSW index (`USING hnsw`) if chunk count exceeds ~500k rows, as IVFFlat recall degrades at scale.

## Migrations Strategy

- Use `prisma migrate dev` for local development.
- Use `prisma migrate deploy` in CI/CD (GitHub Actions) before app boot.
- Never edit migration files after they are committed — always create a new migration.
- Store the database URL in environment variable `DATABASE_URL`.

## Redis (Memory Buffer — M4)

Not managed by Prisma. Used as a sliding-window buffer for conversation context.

| Key pattern | TTL | Value |
|-------------|-----|-------|
| `conv:{conversation_id}:buffer` | 24h | JSON array of last N messages |
| `conv:{conversation_id}:summary` | 7d | Compressed summary of older turns |

Connection via `ioredis`. Single Redis instance is fine for this scale.
